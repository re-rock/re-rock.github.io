We need you!

Everyone is welcome to contribute to the Slim Framework. This project is open source, and it is made better in every possible way thanks to contributors like you.

View Source Code
Reporting Security Issues

If you think that you have found a security issue with the Slim Framework, don't use the support forum or issue tracker and don't publish the issue publicly. Instead, email all security issues to security [at] slimframework.com.

How Can You Help?

There's always something that needs fixed or improved. You can find a list of outstanding tasks on the GitHub Issue Tracker. Browse the open issues and start a discussion on the appropriate threads if you want to help.

If you don't know how to write code, don't worry! You can help improve documentation. You can even fix typos. Don't feel like you have to be a genius developer to help. Anything you can do is greatly appreciated!

Contributor Guidelines

All developers are different, and this is why the Slim Framework project has contributor guidelines. These ensure all contributors follow the same rules and the same code style.

  1. Write PSR-2 compliant code
  2. Use UNIX line returns
  3. Remove trailing white space
  4. Use 4 spaces instead of tabs

Pull Requests

Separate each new feature or improvement into a separate branch in your forked repository. Submit a pull request from each feature branch to the Slim Framework repository's develop branch. All changes should include appropriate PHPUnit tests.
